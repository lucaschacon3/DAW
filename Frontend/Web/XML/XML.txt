<!-- Ejemplo de un catálogo de libros en XML -->

<!DOCTYPE catalogo [
    <!-- Definición de los elementos permitidos en el documento -->
    <!ELEMENT catalogo (libro*)> <!-- El elemento "catalogo" puede contener cero o más elementos "libro" -->
    <!ELEMENT libro (titulo, autor, año, editorial?)> <!-- El elemento "libro" debe contener "titulo", "autor" y "año", y puede contener opcionalmente "editorial" -->
    <!ELEMENT titulo (#PCDATA)> <!-- El elemento "titulo" contiene datos de texto -->
    <!ELEMENT autor (#PCDATA)> <!-- El elemento "autor" contiene datos de texto -->
    <!ELEMENT año (#PCDATA)> <!-- El elemento "año" contiene datos de texto -->
    <!ELEMENT editorial (#PCDATA)> <!-- El elemento "editorial" contiene datos de texto -->
    <!ATTLIST libro
        ISBN CDATA #REQUIRED
        idioma CDATA #IMPLIED
    > <!-- Definición de atributos para el elemento "libro" -->
]>

<!-- Elemento raíz -->
<catalogo>
    <!-- Primer libro -->
    <libro ISBN="978-84-450-0179-7" idioma="es">
        <titulo>El Señor de los Anillos</titulo>
        <autor>J.R.R. Tolkien</autor>
        <año>1954</año>
        <editorial>Minotauro</editorial>
    </libro>

    <!-- Segundo libro -->
    <libro ISBN="978-3-16-148410-0" idioma="es">
        <titulo>Cien años de soledad</titulo>
        <autor>Gabriel García Márquez</autor>
        <año>1967</año>
        <!-- No se especifica la editorial -->
    </libro>

    <!-- Tercer libro -->
    <libro ISBN="978-0-547-05257-9" idioma="en">
        <titulo>1984</titulo>
        <autor>George Orwell</autor>
        <año>1949</año>
        <editorial>Secker and Warburg</editorial>
    </libro>
</catalogo>

<!-- 
    OPERADORES

    *: Cero o más repeticiones.
    +: Una o más repeticiones.
    ?: Cero o una repetición.
    |: Opciones entre elementos.
    ,: Secuencia de elementos.
    &: Intersección de elementos.
    !: Negación de un conjunto de elementos.
    #PCDATA: Define datos de caracteres analizables.
    ANY: Indica cualquier elemento.
    EMPTY: Define un elemento sin contenido

-->

<!-- 
    ATRIBUTOS

    Por defecto: <!ATTLIST ejemplo color CDATA “rojo”>
    #REQUIRED: el atributo es obligatorio, pero no se especifica ningún valor predeterminado.
    #IMPLIED: el atributo no es obligatorio y tampoco se especifica un valor predeterminado
    #FIXED valor: el atributo puedo o no existir, pero si existe su valor es fijo
    CDATA: el atributo contiene caracteres, es decir,texto y sin ningún tipo de restricción
    NMTOKEN: El valor del atributo puede contener letras, dígitos, ".", "-", "_", ":" 
    NMTOKENS:  igual que NMTOKEN pero puede también contener espacios en blanco
    lista de valores: el atributo sólo puede contener uno de los términos de una lista. La lista se escribe entre paréntesis, 
        con los términos separados por la barra "|",  (Bizzum|Visa|4B)

    ID:el valor del atributo (no el nombre) debe ser único y no se puede repetir en otros elementos o atributos. Además el 
        valor no puede empezar por un número o símbolo. Todo atributo declarado de tipo ID tiene que ser #IMPLIED (opcional) o 
        #REQUIRED (obligatorio).
    IDREF:  el valor del atributo debe coincidir con el valor del atributo ID de otro elemento.
    IDREFS: el valor del atributo es una serie de valores separados por espacios que coinciden con el valor del atributo ID de otros elementos
    ENTITY: el valor del atributo es alguna entidad definida en la DTD.
    ENTITIES: el valor del atributo es alguna de las entidades de una lista de entidades definida en la DTD.
    NOTATION: el valor del atributo es alguna notación definida en la DTD.
-->
